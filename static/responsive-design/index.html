Embedding manifest file

To each website, manifest.json file should be added to website to provide additional metadata, such as theme color, name or short name.

Sample file:

{
"short_name": "Clearleft",
"name": "Clearleft design agency",
"start_url": "/",
"background_color": "#00D494",
"theme_color": "#00D494",
"display": "standalone"
}

Accessibility (a11y)

There are number of difficulties that people might have, tritanopia (no blue color). In order to do that, there are couple of good practices to keep in mind, to make the site more accessible:

  1. ARIA - accessible rich internet applications - it provides vocabulary to provide semantic information when there is not a suitable HTML element available, in other words - if we are creating new UI elements, which are not yet available in HTML, we need to keep in mind ARIA standards,
  2. check with different visions types - in Firefox there's "accessibility tab", Chrome DevTools have "emulate vision deficiencies",
  3. when using anchor elements (links), do not rely solely on color, but also on different font weight or text decoration (such as underline),
  4. do not specify color of the element only, but also specify background color together, to assure correct contrast,
  5. adjust to different contrast modes, there's media feature query prefers-contrast, which can take three values: no-preference, less, more. This can be set in the browser, or even through system settings. Also colors might be inverted.
  6. in order to mimic different font-size, it also can be set in system settings or in browser settings,
  7. page can be tested by setting page zoom to 200% or less than 100%,
  8. use tab to navigate through webpage, if it has good UX, to test for people using only keyboard (Firefox offers tool to show tabbing order),
  9. adjust to reduced motion preferences, there's media feature query prefers-reduced-motion, which allows detecting when user prefers reduced motion,
  10. voice - make sure website is well-written also for screen readers (that blind people might use),
  11. use meaningful HTML elements, don't use <div> everywhere, instead use <h1>, ..., <h6>, <main>, <nav>, <aside>, <header>, <footer>, also use <label> element,
  12. always provide alt attirbute to img elements, even empty attribute means "presentational picture", when non specified, means nothing
