Using correct CSS instead of media queries

If we would have to create UI that displays cards, that cannot be wider than 15em, we could try to write media queries with many breakpoints, at 15em, 30em and so on, to account for all display outputs, but instead we also can define correctly grid properties alone: grid-template-columns: repeat(auto-fill, minmax(15em, 1fr)).

Summary

By applying some smart rules in flexbox or grid, it's possible to design dynamic macro layouts with minimal CSS and without any media queries. That's less work for you—you're making the browser do the calculations instead. To see some examples of modern CSS layouts that are fluid without requiring media queries, see 1linelayouts.com.
