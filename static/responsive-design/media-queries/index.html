Media queries

Media queries allow developer to target different types of output. Default is all. Both are equivalent:

body { ... } @media all { body { ... } }

To target different outputs, we can do it by specifying link media attribute:

<link rel="stylesheet" href="print.css" media="print" />

Or directly in CSS:

@media print { body { ... }}

Query conditions

Also conditions can be added to medtia types. They are called media queries. Syntax is:

@media type and (feature)

Media queries can also be used on a link element.

<link rel="stylesheet" href="specific.css" media="type and (feature)">

For example, orientation media feature is used to apply styles based on whether window is in landscape mode (the viewport is greater than its height) or portrait mode (the viewport height is greater than its width).

In landscape mode, I have different background than in portrait mode.

And I will have first letter nicely styled, if the viewport width exceeds 400px.

To prevent browser from scaling the page on different screen sizes, we need to use below tag:

<meta name="viewport" content="width=device-width, initial-scale=1">

Media types

These are optional in media query.

There are only four:

  • all (default)
  • screen
  • print
  • speech

Media features

This are required in media query.

These are lots of them and they have expaned greatly over the years.

  • viewport dimensions
    • min-width, max-width, min-height, max-height which can take numeric values, such as 250px,
    • min-aspect-ratio, max-aspect-ratio, aspect-ratio which can be compared against, for example, 16/9,
    • orientation, which can take value portrait or landscape
  • displays
    • min-resolution, max-resolution, resolution - can be compared against, for example, 400dpi (dots per inch).
    • update - which is refresh rate of the device, can take one of the values: none, slow, fast,
    • display-mode whcih can take values fullscreen, standalone, minimm-ui, browser (value comes from manifest file),
  • color
    • monochrome - usage @media (monochrome) - true if the device only outputs shades of gray
    • color - usage: @media (color) - true if the device displays all colors,
    • color-gamut, color-index, dynamic-range - all of them report specific details about the color capabilities of the screen,
  • interaction
    • hover and any-hover - which can take values hover and none
    • pointer and any-pointer - which can take values fine, coarse and none
  • preferences
    • prefers-color-scheme
    • prefers-contrast
    • prefers-reduced-motion
  • more media features
    • forced-colors - tells if the device uses restricted colors pallette
    • inverted-colors - tells if the device uses inverted colors pallette
    • scripting - tells if JavaScript is enabled
    • prefers-reduced-data - tells if users prefer less bandwidth (network traffic), so the page could load, for example, less images,
