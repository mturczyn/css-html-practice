div:first-child is defined to have bold font, but is not applied, as div is not first child.

But p has defined before pseudo class matching p:first-child selector, so it has checkmark, as it p is first child.

Lastly, there is red color defined for div:first-of-type, which is applied to first child, which has type of div

Similairly, we have :last-of-type, which works analogically to its counterpart :first-of-type, as can be just observed (here we have last child of type code).

Similairly we have nth-child() and nth-of-type() pseudo classes to match exactly elements at specified index.

For example, here we have list of words wrapped in code elements (C# keywords): using, class, interface, record. Here we used code:nth-of-type(2).

Interesting is use of even and odd keywords to match alternating elements, for eample, using the same list of C# keywords: using, class, interface, record, public, private, protected, sealed, get, set. Every even occurence of code element gets style.

You can also specify the An+B microsyntax, which allows to specify pattern, such as 3n+2, which will match every third item and start at second item. Using the same list of C# keywords: using, class, interface, record, public, private, protected, sealed, get, set.
